# Garvit Portfolio — Setup Guide

## Files
- index.html   → main page
- style.css    → all styles
- auth.js      → Google login via Firebase
- contact.js   → contact form via EmailJS

## How to run locally
Just open index.html in a browser. For Firebase login to work locally, add `localhost` to Firebase authorized domains.

## How to host (Netlify — free & easy)
1. Go to https://netlify.com → Sign up free
2. Drag and drop this entire folder onto the Netlify dashboard
3. Your site goes live instantly with a free URL
4. Copy the URL (e.g. yoursite.netlify.app)
5. Go to Firebase Console → Authentication → Settings → Authorized Domains → Add that URL

## Firebase Security Rules (important!)
In Firebase Console → Firestore/Realtime DB → Rules, make sure only authenticated users can write:
  allow read, write: if request.auth != null;

## Are the API keys safe?
Yes. Firebase and EmailJS keys in frontend JS are PUBLIC identifiers — they are meant to be visible.
Real security comes from:
- Firebase Authorized Domains (only YOUR site can use your Firebase)
- Firebase Security Rules (control who can read/write data)
- EmailJS domain whitelist (set in EmailJS dashboard → Account → Security)

## Add social links later
In index.html, find the <a href="..."> tags in the SOCIALS section and replace the URLs.
